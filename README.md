# AI CS Helper

대한항공 고객 문의를 AI가 1차로 분석·응답하고, 상담원이 최종 검토·승인하는 **AI 기반 항공사 고객상담 보조 시스템**입니다.

---

## 1. 프로젝트 소개

AI CS Helper는 항공사 고객센터에 접수되는 반복적인 규정 문의(수하물, 환불수수료, 마일리지, 예약 변경 등)를 **RAG(검색 증강 생성) + LLM 파이프라인**으로 자동 분석하고 답변 초안을 생성하는 서비스입니다.

- AI가 규정 문서를 검색해 근거 있는 답변을 초안으로 작성하고, 스스로 그 답변을 한 번 더 검수합니다.
- 최종 승인/수정 권한은 항상 상담원에게 있습니다 — AI는 초안을 제안할 뿐, 고객에게 직접 발송되는 답변은 상담원 관리자 페이지에서 승인된 것만 나갑니다.
- 문의가 들어온 순간부터 최종 답변이 등록될 때까지 전 과정(마스킹 → 검색 → 판단 → 검수 → 저장)이 로그로 남아, 상담원이 "AI가 왜 이렇게 판단했는지"를 그대로 재생해볼 수 있습니다.

## 2. 문제 정의

- **반복 업무 과부하**: 수하물 허용량, 환불수수료, 마일리지 등 규정만 알면 답할 수 있는 문의가 상담원의 시간을 크게 차지합니다.
- **응답 지연**: 사람이 매번 규정 문서를 찾아 확인하고 답변을 작성해야 해서, 간단한 문의도 처리까지 시간이 걸립니다.
- **개인정보 노출 위험**: 문의 내용에는 이름, 전화번호, 결제정보 등 개인정보가 섞여 들어오는 경우가 많은데, 이를 외부 AI 서비스에 그대로 넘기면 안 됩니다.
- **AI 자동화의 신뢰 문제**: AI가 잘못된 답변(환각, 규정 오적용)을 그대로 고객에게 보내면 신뢰 문제로 직결됩니다. 무조건 자동화하기도, 무조건 사람이 다 처리하기도 어려운 딜레마가 있습니다.

## 3. 문제 해결

| 문제 | 해결 방식 |
|---|---|
| 반복 업무 과부하 | 규정 문서를 임베딩해 RAG로 검색하고, LLM이 관련 규정을 근거로 답변 초안을 자동 생성 |
| 응답 지연 | 문의 접수 → 마스킹 → 검색 → 답변 생성까지 파이프라인이 자동으로 처리, 상담원은 검토·승인만 수행 |
| 개인정보 노출 | 정규식 기반 마스킹을 거쳐 이메일·주민등록번호·전화번호·카드번호 등을 치환한 뒤에만 외부 AI(임베딩/LLM)로 전달 |
| AI 신뢰 문제 | AI 답변을 별도의 AI 검수 단계가 pass/fail로 재평가(최대 3회 재작성), 그래도 못 미더우면 사람에게 이관. 최종 발송은 항상 상담원 승인을 거침 |
| 장애 대응 | 임베딩/LLM 호출이 실패해도 파이프라인 전체가 죽지 않고 "상담원 확인 필요" 상태로 안전하게 이관 |

## 4. 핵심 기능

- **AI 우선 답변 생성**: 규정 근거가 조금이라도 있으면 AI가 먼저 답변을 시도하고, 정말 근거가 없을 때만 상담원에게 이관합니다.
- **규정 기반 RAG 검색**: 대한항공 규정 문서를 임베딩해 저장해두고, 문의와 코사인 유사도가 높은 상위 문서를 근거로 활용합니다.
- **개인정보 자동 마스킹**: 이메일/주민등록번호/카드번호/전화번호/이름/주소/계좌번호/여권번호를 정규식으로 탐지해 치환 후 외부로 전달합니다.
- **AI 셀프 리뷰 루프**: 생성된 답변의 타당성을 별도 LLM이 검수하고, 반려되면 사유를 반영해 최대 3회까지 답변을 재작성합니다.
- **상담원 관리자 페이지**: 전체 처리내역 / 처리대기 / 통계 3개 탭으로 문의를 관리하며, AI 답변을 승인·수정하거나 직접 작성할 수 있습니다.
- **AI 처리 타임라인 시각화**: 마스킹 → RAG 검색 → 라우팅 → 검수 → DB 저장까지 전 과정을 문의별로 순서대로 재생해 볼 수 있습니다.
- **통계 대시보드**: 전체 문의 수, AI 자동응답률, 처리대기 건수 KPI와 카테고리별·상태별·답변자별 분포 차트를 제공합니다.
- **RAGAS 기반 품질 평가**: 상담원이 승인한 최종 답변을 정답 삼아 AI 초안의 품질(환각 여부, 검색 정확도, 답변 정확도 등)을 정량적으로 평가하는 도구를 제공합니다.
- **자동 배포 파이프라인**: GitHub Actions로 lint·테스트를 자동 검증하고, main 브랜치 반영 시 GCP VM에 자동 배포됩니다.
